# Boas Práticas para Git e GitHub

1. **Sempre adicionar mensagens claras nos commits**: Use mensagens descritivas que expliquem o que foi alterado, facilitando o entendimento do histórico.

2. **Organizar branches**: Crie branches separadas para cada feature ou correção, evitando conflitos na branch principal.

3. **Fazer commits pequenos e frequentes**: Commite mudanças relacionadas em grupos pequenos, tornando o histórico mais granular e fácil de revisar.
